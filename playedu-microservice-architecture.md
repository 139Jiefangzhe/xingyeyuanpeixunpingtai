# PlayEdu 二次开发微服务架构设计方案

> **版本**：v1.0  
> **定位**：基于 PlayEdu 开源版，向企学星级别的企业内训平台演进  
> **约束**：内部使用、单租户、微服务部署、Java + React 技术栈统一

---

## 一、设计目标与核心约束

| 维度 | 目标/约束 |
|------|----------|
| **功能对标** | 覆盖企学星核心能力：课程中心、考试测练、直播会议、培训任务、学习社区、积分运营、人才发展、统计分析 |
| **技术基座** | 以 PlayEdu 开源版（Java 17 + Spring Boot 3 + React 18 + MySQL）为基座，保留其部门/学员/视频课程核心骨架 |
| **部署形态** | 内部私有化部署，无需多租户隔离，但需保留组织架构与 RBAC 权限体系 |
| **架构风格** | 领域驱动拆分微服务，前后端分离，共享基础设施 |
| **非功能性** | 支持 2000+ 并发在线学习，核心接口 P99 < 500ms，视频首帧 < 2s |

## 三、服务拆分矩阵（10 个核心服务）

| 服务名 | 英文标识 | 基座来源 | 核心职责 | 二开工作量 |
|--------|---------|---------|---------|-----------|
| **用户中心** | `user-svc` | PlayEdu 用户/部门/权限模块 | 组织架构、用户管理、角色权限、SSO/OAuth2、LDAP/企微/钉钉对接 | 中（扩展第三方登录） |
| **课程中心** | `course-svc` | PlayEdu 课程/资源模块 | 课程/章节/分类、视频课/文档课、学习进度追踪、SCORM 适配器、收藏/评价 | 低（保持现有） |
| **考试中心** | `exam-svc` | **新增** | 题库管理（单选/多选/判断/填空/问答）、智能组卷、在线考试、防作弊、自动评分、成绩分析 | 高（完全新建） |
| **直播中心** | `live-svc` | **新增** | 直播间管理、推流/拉流配置、弹幕/问答/签到/白板、录制转存、课程关联 | 高（完全新建） |
| **培训中心** | `train-svc` | **新增** | 培训项目、学习任务包、O2O 混合培训、师徒制、实操作业、审批流、新员工培训路径 | 高（完全新建） |
| **学习社区** | `community-svc` | **新增** | 论坛板块、帖子/问答、评论/点赞/举报、话题/圈子、精华/置顶 | 中（参考南生论坛） |
| **积分商城** | `point-svc` | **新增** | 积分账户、积分获取规则（学习/考试/签到）、积分商城、兑换记录、商品管理 | 中 |
| **人才发展** | `talent-svc` | **新增** | 岗位能力模型、学习地图/路径、岗位认证、证书模板、晋级考核 | 中 |
| **统计分析** | `stats-svc` | **新增** | 学习时长统计、课程完成率、考试分析、人才画像、数据大屏、报表导出 | 中 |
| **文件与消息** | `file-svc` / `msg-svc` | PlayEdu 资源模块扩展 | 文件上传/分片/转码/水印、站内通知、短信/邮件/企微推送 | 低 |

> **策略**：PlayEdu 原有的用户、课程、视频、权限核心代码直接迁移到对应服务作为 `v1.0` 基线；新增服务独立开发，通过 REST + MQ 与基座服务协作。

## 四、核心模块详细设计

### 4.1 用户中心 (user-svc)

**定位**：全局身份与权限基础设施，所有服务的统一认证入口。

| 功能域 | 说明 |
|--------|------|
| **组织架构** | 无限级部门树、岗位、职级；支持 Excel 批量导入（复用 PlayEdu 能力） |
| **用户管理** | 学员/讲师/管理员三角色；用户标签体系（岗位/技能/入职时间） |
| **认证体系** | JWT AccessToken + RefreshToken；密码登录/验证码登录/扫码登录（企微/钉钉/飞书） |
| **权限模型** | RBAC：角色-菜单-按钮三级；数据权限（部门可见范围/全公司/仅自己） |
| **第三方同步** | LDAP/AD 同步、企微/钉钉通讯录同步（事件驱动，定时 + 手动） |

**核心实体**：
```
Dept(部门) → User(用户) → Role(角色) → Permission(权限)
     ↑          ↑              ↑
  DeptUser   UserRole      RolePermission
```

**对外 API**：
- `GET /api/users/{id}` — 查询用户基础信息（供其他服务调用）
- `GET /api/users/departments/{deptId}/members` — 部门成员查询
- `POST /api/auth/verify` — Token 校验（Gateway 层调用）
- `POST /api/auth/permissions/check` — 权限校验

---

### 4.2 课程中心 (course-svc)

**定位**：内容资产管理 + 学习过程追踪，PlayEdu 原有能力的服务化。

| 功能域 | 说明 |
|--------|------|
| **课程管理** | 章节课（固定顺序）、自由课（自选）、文档课（PDF/Word/PPT 在线预览） |
| **资源存储** | 视频：对接 MinIO/COS + CDN；文档：转 PDF 后预览 |
| **学习记录** | 视频播放进度记忆续播、防快进（配置化开关）、学习时长实时上报 |
| **SCORM 支持** | 新增 SCORM 1.2 / xAPI 运行时环境（前端 iframe + 后端 CMI 数据接口） |
| **课程运营** | 分类标签、难度等级、收藏、评分、评论（与社区服务联动） |

**核心实体**：
```
Course → Chapter → Lesson(Resource)
   ↓         ↓         ↓
CourseStudent    LearningRecord    LearningProgress
```

**关键事件（MQ Topic: `course.event`）**：
- `CourseStarted` — 课程开始学习
- `LessonCompleted` — 课节完成
- `CourseCompleted` — 课程完成（触发积分/培训任务校验）

---

### 4.3 考试中心 (exam-svc)

**定位**：覆盖从题库建设到成绩分析的全链路考试能力。

| 功能域 | 说明 |
|--------|------|
| **题库管理** | 题型：单选/多选/判断/填空/问答/组合题；支持 LaTeX 公式、图片、音频插入；题库分类与权限 |
| **组卷策略** | 手动选题 / 智能随机抽题（按题型/难度/知识点/数量规则）；A/B 卷 / 乱序 |
| **考试发布** | 考试时间窗口、时长限制、参考人员范围（部门/岗位/指定人）、及格线、重考次数 |
| **防作弊机制** | 切屏次数监控、IP 校验、人脸识别（预留接口）、题目乱序、选项乱序 |
| **阅卷评分** | 客观题自动评分；主观题分配阅卷人、多人复阅、评分仲裁 |
| **成绩与证书** | 成绩分析（最高分/最低分/平均分/及格率/分数段分布）；个人成绩单导出 PDF |

**核心实体**：
```
QuestionBank → Question (多种题型子表)
     ↓
ExamPaper → PaperQuestion → ExamSession → ExamRecord → ExamResult
```

**数据分片建议**：`exam_record` 按考试 ID 或用户 ID 分表（考试记录量大）。

---

### 4.4 直播中心 (live-svc)

**定位**：轻量化直播管理服务，流媒体能力依赖云厂商，自身负责业务编排。

| 功能域 | 说明 |
|--------|------|
| **直播间管理** | 创建直播间（推流地址/拉流地址/录制配置）、关联课程/培训项目、讲师权限 |
| **直播互动** | 弹幕（IM 通道）、问答（问答墙）、签到（位置/扫码）、在线人数 |
| **直播观看** | 多线路切换（超清/高清/流畅）、移动端自适应、画中画 |
| **录制回放** | 直播结束自动转存 VOD，生成回放课程，关联到课程中心 |
| **直播统计** | 观看人数峰值、人均观看时长、互动频次 |

**架构策略**：
- **推流/拉流/录制**：腾讯云直播 / 阿里云直播 SaaS（或 SRS 私有化）
- **IM/弹幕/信令**：腾讯云 IM 或自研 WebSocket 集群（`live-websocket-svc` 可选）
- **业务数据**：直播间元数据、互动记录、签到记录存 MySQL

---

### 4.5 培训中心 (train-svc)

**定位**：面向企业培训场景的项目化管理，支持多种培训模式。

| 功能域 | 说明 |
|--------|------|
| **培训项目** | 创建培训计划 → 添加课程/考试/直播/作业作为任务节点 → 指定学员范围 → 设置完成规则 |
| **学习任务** | 任务派发、学习期限、完成标准（看完/看完+考试通过/积分达标） |
| **O2O 混合** | 线上课程 + 线下签到 + 线下考试组合；支持地理位置签到 |
| **师徒制** | 导师-学员绑定、带教任务下发、导师评价、学员出师考核 |
| **实操作业** | 作业模板、文件提交、评审打分、退回重做 |
| **新员工培训** | 入职自动化触发（对接 HR 系统或手动分配）、固定路径（Day1→Day7→Day30） |
| **审批流** | 培训申请审批、作业提交审批（轻量级工作流引擎：自研状态机或集成 Camunda） |

**状态机示例（培训项目）**：
```
Draft → Published → InProgress → Completed → Archived
              ↓        ↓
         Paused    Cancelled
```

### 4.6 学习社区 (community-svc)

**定位**：社交化学习，激活 UGC 与互动氛围。

| 功能域 | 说明 |
|--------|------|
| **内容形态** | 帖子（图文/视频）、问答（悬赏积分）、文章（长文/知识沉淀） |
| **互动机制** | 点赞、收藏、评论（嵌套回复）、转发、举报 |
| **圈子/话题** | 按部门/岗位/兴趣创建圈子；话题标签聚合 |
| **内容审核** | 敏感词过滤 + 人工审核后台；精华/置顶/加精 |
| **与课程联动** | 课程讨论区自动创建对应板块；学完课程引导发帖 |

**技术参考**：表结构可借鉴南生论坛（`post`、`comment`、`circle`、`user_action`）。

---

### 4.7 积分商城 (point-svc)

**定位**：游戏化运营引擎，驱动学习行为。

| 功能域 | 说明 |
|--------|------|
| **积分规则引擎** | 配置化规则：完成课程 +10、考试通过 +20、每日签到 +5、社区发帖 +3；支持条件组合 |
| **积分账户** | 总积分、可用积分、冻结积分、过期策略；流水记录不可篡改 |
| **商城商品** | 虚拟商品（课程解锁/证书加速/头衔）/ 实体商品；库存管理 |
| **兑换与发放** | 兑换申请 → 审核（可选）→ 发放 → 物流/系统到账；支持抽奖/转盘 |
| **排行榜** | 个人周榜/月榜、部门榜；Redis Sorted Set 实时计算 |

**事件订阅**：订阅 `course.event`、`exam.event`、`community.event`，异步发放积分。

---

### 4.8 人才发展 (talent-svc)

**定位**：岗位能力模型与员工成长路径。

| 功能域 | 说明 |
|--------|------|
| **岗位模型** | 岗位 → 能力维度 → 技能点 → 关联课程/考试/证书；胜任力雷达图 |
| **学习地图** | 新员工路径 / 晋升路径 / 转岗路径；节点解锁机制（前置条件） |
| **岗位认证** | 认证标准配置、申请入口、审核流程、电子证书生成与验真 |
| **人才画像** | 聚合课程完成率、考试分数、技能覆盖度、学习时长 → 生成能力档案 |

---

### 4.9 统计分析 (stats-svc)

**定位**：数据中台与决策支持，只读 + 聚合计算。

| 功能域 | 说明 |
|--------|------|
| **实时看板** | 当前在线人数、今日学习人数、课程播放量（Redis + 定时落库） |
| **学习报表** | 部门学习完成率、人均学习时长、课程热度 Top10、考试及格率 |
| **个人档案** | 学员全生命周期学习轨迹时间轴 |
| **数据导出** | 报表定时生成 + 异步下载；支持 Excel/PDF |
| **数据大屏** | 可选：对接 DataV / FineBI / 自研大屏 |

**数据流**：
```
业务库 MySQL → Canal/Debezium → Kafka → ClickHouse/ES → stats-svc 聚合查询
```

## 五、数据层架构

### 5.1 数据库拆分策略

采用**按服务分库**（Database per Service），避免单库瓶颈。

| 服务 | 数据库 | 核心表 | 备注 |
|------|--------|--------|------|

### 5.2 缓存策略

| 数据类型 | 缓存方案 | TTL |
|---------|---------|-----|
| 用户 Token / 会话 | Redis Hash | 2h |
| 用户基础信息 | Redis String + 本地 Caffeine | 30min / 5min |
| 部门树 / 权限菜单 | Redis String | 1h |
| 课程热度 / 排行榜 | Redis Sorted Set | 实时计算，10min 持久化 |
| 考试防重复提交 | Redis SETNX | 考试期间 |
| 验证码 / 限流计数 | Redis String | 5min |
| 视频防盗链 Token | Redis String | 1h |

### 5.3 搜索与对象存储

- **Elasticsearch**：课程全文检索（标题/描述/讲师）、社区帖子搜索、用户检索
- **MinIO / 腾讯云 COS**：视频、文档、图片、证书模板统一对象存储
- **CDN**：视频 HLS 切片、静态资源、课程封面图加速

## 六、基础设施层设计

### 6.1 API 网关 (Gateway)

| 职责 | 实现 |
|------|------|
| **动态路由** | 基于 Nacos 服务发现，自动注册/摘除 |
| **统一鉴权** | JWT 校验 → 调用 user-svc 鉴权接口 → 写入请求头 `X-User-Id` |
| **限流熔断** | Sentinel：接口 QPS 限流、慢调用降级、异常比例熔断 |
| **灰度发布** | Gateway Filter + Nacos 元数据权重路由 |
| **日志追踪** | 生成 `Trace-ID`，写入 MDC，全链路传递 |

### 6.2 服务注册与配置中心 (Nacos)

- **注册中心**：所有服务启动时注册，Gateway 动态发现
- **配置中心**：`shared-config.yml`（公共配置）+ `{service}-config.yml`（服务私有配置），支持热更新
- **命名空间**：`dev` / `test` / `prod` 环境隔离

### 6.3 监控与可观测性

| 组件 | 用途 |
|------|------|
| **SkyWalking** | 分布式链路追踪，排查跨服务慢调用 |
| **Prometheus + Grafana** | 服务 QPS/RT/错误率/JVM 指标可视化大盘 |
| **ELK / Loki** | 日志采集、检索、告警（错误日志 > 10/min 触发告警） |
| **Sentinel 控制台** | 实时限流规则调整、热点流量监控 |

### 6.4 消息队列 (RocketMQ)

| Topic | 生产者 | 消费者 | 用途 |
|-------|--------|--------|------|
| `course-learning` | course-svc | point-svc, stats-svc, train-svc | 学习完成事件 |
| `exam-result` | exam-svc | point-svc, stats-svc, talent-svc, msg-svc | 考试结果通知 |
| `live-start` | live-svc | msg-svc, course-svc | 直播开始提醒 |
| `file-transcode` | file-svc | course-svc | 转码完成回调 |
| `point-change` | point-svc | stats-svc | 积分变动流水 |
| `train-remind` | XXL-Job | msg-svc | 培训到期提醒 |

## 七、前端架构整合

### 7.1 前端应用划分

采用**单页应用聚合**而非微前端（降低复杂度），统一使用 React 18 + Ant Design 5。

| 应用 | 对应后端 | 说明 |
|------|---------|------|
| `edu-pc` | Gateway → 聚合 API | 学员学习端（PC）：课程学习、考试、直播、社区、个人中心 |
| `edu-h5` | Gateway → 聚合 API | 学员移动端（H5）：适配手机浏览器，核心功能同 PC |
| `edu-admin` | Gateway → 聚合 API | 管理后台：所有模块的后台管理统一入口，基于 RBAC 动态菜单 |
| `edu-lecturer` | Gateway → 聚合 API | 讲师工作台：课程上传、直播开播、作业批改、答疑 |

### 7.2 BFF 聚合层（可选）

若前端调用多次后端接口，可引入 `edu-bff-svc`（Node.js / Java）做接口聚合：
- 学员首页：一次请求返回「在学课程 + 待考试 + 最新公告 + 积分」
- 管理首页：一次请求返回「今日数据 + 待审批 + 系统通知」

### 7.3 与 PlayEdu 前端的关系

- **基座复用**：PlayEdu 原有 React 项目代码作为 `edu-admin` 和 `edu-pc` 的 `v1.0` 基座
- **渐进重构**：新增页面按新规范开发；旧页面逐步迁移至 Ant Design 5 统一设计语言
- **构建改造**：从单体式构建改为按应用独立打包、独立部署

## 十一、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| PlayEdu 开源版协议限制 | 无法去除版权 | 保留页脚版权信息，后台标注 "Powered by PlayEdu"；核心新增模块代码完全自有 |
| 微服务拆分过度导致延迟 | 排期翻倍 | Phase 1 先拆 2 个核心服务，其余模块可先用单体jar内嵌启动，后续再拆 |
| 视频直播延迟高 | 体验差 | 直播走 CDN + HLS，延迟 3-5s 可接受；强互动场景预留 WebRTC 升级空间 |
| 考试并发压力大 | 系统卡顿 | 考试开始时间错峰 + 题库/试卷缓存预热 + Redis 限流防刷 |
| 数据一致性问题 | 积分/学习记录对不上 | 核心业务（考试提交、培训完成）用本地事务；积分发放用 MQ 最终一致 + 对账补偿 |

> **总结**：本方案以 PlayEdu 为最小可行基座，通过微服务拆分将原有能力服务化，并在其周围生长出考试、直播、培训、社区、积分、人才六大新域。基础设施统一采用 Spring Cloud Alibaba 生态，部署于 Kubernetes，预期 16 周内完成从基座到完整企业内训平台的演进。
